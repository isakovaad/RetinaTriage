## What was done

- wrote the baseline training script: a resnet50 from `timm`, class-weighted loss to deal with the imbalance, quadratic weighted kappa as the metric (the right one for ordinal DR grades), and mixed precision.
- trained 5 epochs. the in-domain validation kappa climbed every single epoch — 0.78 → 0.80 → 0.84 → 0.86 → **0.87**. saved the best model and got the `.pt` onto my laptop.
- built the fastapi backend that loads the `.pt` and serves a `/predict` endpoint. it reuses the *exact* same preprocessing as training, so the model sees images the same way it did when it learned — keeps the live accuracy honest.
- got the backend running locally and tested it end to end: uploaded a fundus image and got back grade 2 (moderate), `refer: true`, with the probability mass sitting on grades 2/3/4 and basically nothing on "no DR". so the whole upload → grade slice works now.
- from the mentor sessions i applied a few things. fine-tuning a pretrained model instead of training from scratch (s1 — this is the transfer-learning version of the fine-tuning idea). the verification-ladder / "don't trust, instrument" mindset (s1): quad-weighted kappa as an honest metric, a held-out validation split, class-weighting, and making the backend reuse the exact training preprocessing so serving matches training. and the incremental build habit from s5 — data pipeline → baseline → backend, checking each step actually works before moving to the next, like the "stop and show me after every endpoint" rule.

## Problems encountered

- first training run was silently on **cpu** (~24s per batch, basically unusable) — didn't realize the gpu accelerator is a separate setting from the internet toggle.
- switched to gpu but picked the **P100**, which crashed with "no kernel image available for execution on the device" — the pytorch build had dropped support for that card's older architecture.
- kaggle wipes `/kaggle/working` on every restart / accelerator switch, so i lost the processed data and the trained model a couple of times.
- the backend wouldn't start — `ModuleNotFoundError: No module named pandas`.
- the kaggle `FileLink` download just showed "not found" when i tried to pull an image out.

## Solutions

- enabled the gpu accelerator (separate setting!), then switched **P100 → T4**, which is supported. training dropped to ~35s per *epoch*.
- learned to set the accelerator first, run top to bottom, and download the `.pt` + save a version straight away so a wipe doesn't cost me the model.
- installed pandas in the backend venv — the inference code imports a module that pulls it in.
- skipped `FileLink` entirely and just displayed the image inline in the notebook, then right-clicked and saved it.
